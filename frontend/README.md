# YellowNews — Frontend

## Стек технологий

| Технология                  | Назначение                                           |
| --------------------------- | ---------------------------------------------------- |
| **Vue 3** (Composition API) | Основной фреймворк для UI                            |
| **TypeScript**              | Строгая типизация и автодополнение кода              |
| **Vite**                    | Сборщик проекта (быстрый HMR и билд)                 |
| **Vue Router**              | Маршрутизация (навигация между страницами)           |
| **Axios**                   | HTTP-клиент для взаимодействия с API                 |
| **Socket.IO Client**        | Уведомления и обновления в реальном времени          |
| **Sass / SCSS**             | Препроцессор CSS для удобной стилизации и переменных |

---

## Особенности и функционал

- **Полная аутентификация:** Регистрация, вход, использование JWT токенов (через Cookies).
- **Защищенные маршруты (Protected Routes):** Автоматические редиректы неавторизованных пользователей на страницу входа. Перехват 401 ошибок.
- **Управление новостями (CRUD):** Создание, чтение, редактирование и удаление статей.
- **Планирование публикаций:** Возможность сохранить статью в черновик (Draft), опубликовать мгновенно (Published) или запланировать на потом (Scheduled).
- **Реактивность UI:** Использование Composable-функций (`useAuth`, `useNews`, `useUpload`) для управления глобальным стейтом приложения.
- **WebSocket уведомления:** Интеграция с бекендом по Socket.IO для мгновенного оповещения о новых новостях.
- **Отзывчивый дизайн (Responsive):** Современный дизайн с использованием SCSS переменных.

## Запуск (Локально)

### 1. Установка зависимостей

Убедитесь, что у вас установлен Node.js. Затем перейдите в директорию `frontend` и выполните:

```bash
cd frontend
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне папки `frontend` или скопируйте его из примера:

```bash
cp .env.example .env
```

Убедитесь, что внутри `.env` указаны правильные адреса для связи с вашим бэкендом:

```env
# URL вашего бэкенда (с которым будут работать запросы Axios)
VITE_API_BASE_URL=http://localhost:5000

# URL для WebSocket подключений (Socket.IO)
VITE_WS_URL=http://localhost:5000
```

> При деплое в интернет (например, на Vercel) вам нужно будет изменить эти ссылки на адрес вашего развернутого бэкенда.

### 3. Запуск в режиме разработчика

```bash
npm run dev
```

Приложение запустится локально по адресу: `http://localhost:5173`.

## Сборка для Production

Для создания оптимизированной production сборки выполните:

```bash
npm run build
```

Скомпилированные статические файлы HTML, CSS и JS будут созданы в папке `dist/`. Эту папку можно разместить на любом статическом хостинге, например, **Vercel**, **Netlify** или обычный **Nginx/Apache**.
